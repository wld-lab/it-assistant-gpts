# Цель проекта

Целью проекта является разработка кастомного ИИ-ассистента для обработки типовых IT-запросов сотрудников компании.

Ассистент позволяет:

* ускорить решение стандартных проблем  
* снизить нагрузку на IT-поддержку  
* повысить удобство взаимодействия сотрудников с IT-сервисами

# Бизнес-задачи

Решаемые задачи:

* автоматизация первой линии поддержки (L1)  
* сокращение времени обработки обращений  
* уменьшение количества пользовательских запросов в IT- службах  
* стандартизация ответов на типовые вопросы

##  Описание ассистента

Ассистент анализирует запрос пользователя и:

1. Определяет тип проблемы  
2. При необходимости уточняет детали  
3. Предлагает пошаговое решение  
4. Проверяет результат  
5. При необходимости направляет в соответствующий отдел

## Архитектура решения

В проекте реализована архитектура, основанная на разделении:

## 1\. База знаний (FAQ)

Предназначена для быстрого стандартизованного ответа  на типовые вопросы пользователей в данной компании.

Содержит:

* формулировки типовых проблем  
* пошаговые инструкции  
* подтипы проблем

Подтип является ключевым элементом, который:

* связывает пользовательский запрос с типом проблемы  
* используется для маршрутизации

Важно: несколько вопросов могут относиться к одному подтипу.

## 2\. Таблица маршрутизации

Содержит:

* подтипы проблем  
* соответствующие отделы  
* контактные данные (email, Telegram, телефон)  
* приоритет

Логика:

FAQ → определяет подтип  
Routing → определяет отдел и контакты

## 3\. Логика работы ассистента

Ассистент работает по следующему алгоритму:

1. Анализ запроса пользователя  
2. Поиск соответствующего кейса в FAQ  
3. Определение подтипа  
4. Предложение решения  
5. При необходимости — эскалация:  
   * определение отдела по подтипу  
   * выдача контактов

**Если информации по запросу пользователя нет в базе знаний, ассистент предлагает решение самостоятельно. Для ИТ- области и современных моделей ИИ это представляется вполне оправданным.**

### Логика эскалации

Эскалация происходит, если:

* предложенные шаги не помогли  
* проблема требует проверки системы  
* пользователь явно просит контакты

При эскалации ассистент:

* указывает конкретный обслуживающий отдел   
* выводит контакты для обращения   
* не генерирует данные по контактам, а использует только информацию из базы знаний (таблица маршрутизации)

### Ограничения

Ассистент:

* не выполняет действия за пользователя  
* не выдает доступы  
* не работает с чувствительными данными  
* не заменяет IT-специалиста
